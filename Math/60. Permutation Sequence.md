# **LeetCode Permutation Sequence**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The problem requires us to find the k-th permutation of a sequence of numbers from 1 to n. The key idea is to recognize that the permutations can be systematically organized and the k-th permutation can be constructed without generating all previous permutations. 

### **🚀 Approach**  
1. **Factorials**: First, we need to calculate the factorials of numbers up to n, as these will help us determine the number of permutations for each position.
2. **Positioning**: For each position in the result, we determine which number can be placed at that position based on how many permutations fit under the k-th sequence.
3. **Adjust k**: After selecting a number, we adjust k to account for the permutations skipped based on the number chosen.
4. **Output**: We continue this process until all positions are filled. 

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n^2) due to the need to remove used elements from the list of available numbers.
- **Space Complexity**: O(n) for storing the result and factorial values.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    string getPermutation(int n, int k) {
        vector<int> factorials(n + 1, 1);
        for (int i = 1; i <= n; ++i) {
            factorials[i] = factorials[i - 1] * i;
        }

        vector<int> numbers;
        for (int i = 1; i <= n; ++i) {
            numbers.push_back(i);
        }

        string permutation;
        k--; // Convert k from 1-based to 0-based index

        for (int i = 1; i <= n; ++i) {
            int index = k / factorials[n - i];
            permutation += to_string(numbers[index]);
            numbers.erase(numbers.begin() + index);
            k %= factorials[n - i];
        }

        return permutation;
    }
};
```
