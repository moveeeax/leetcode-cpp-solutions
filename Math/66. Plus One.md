# **LeetCode Plus One**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The task is to increment a non-negative integer represented as an array of its digits. We need to simulate adding one to the number, handling the cases where the addition might lead to a carry that affects the next digit. The challenge lies in manipulating the array correctly to account for these carries, especially when the most significant digit (the first element of the array) needs to change as well.

### **🚀 Approach**  
1. Start from the last digit of the array (the least significant digit) and add one to it.
2. Check if this addition causes a carry (i.e., if the digit becomes 10). 
3. If there is a carry:
   - Set the current digit to 0.
   - Move to the next more significant digit (left).
   - Repeat the process until there are no more digits left to check or there is no carry overshadowing the digits.
4. If after processing all digits, there remains a carry (which means we incremented from something like `999` to `1000`), we should add a `1` at the beginning of the array.
5. Return the modified array.

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n) where n is the number of digits in the input array. In the worst case, we may need to iterate through all the digits.
- **Space Complexity**: O(1) if we modify the input array in place. However, in some cases, we may create a new array if we need to add an additional digit (e.g., when transforming `999` to `1000`). Hence, O(n) in the case of returning a new array.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    vector<int> plusOne(vector<int>& digits) {
        int n = digits.size();
        for (int i = n - 1; i >= 0; --i) {
            if (digits[i] < 9) {
                digits[i]++;
                return digits;
            }
            digits[i] = 0; // carry over
        }
        // If we finish the loop, it means we had a carry for the most significant digit.
        digits.insert(digits.begin(), 1);
        return digits;
    }
};
```
