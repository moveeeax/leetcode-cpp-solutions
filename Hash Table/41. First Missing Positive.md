# **First Missing Positive**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The goal of this problem is to find the smallest missing positive integer from an unsorted integer array. The missing positive integer is the first positive number that does not appear in the array. This requires an efficient solution, ideally with constant space and linear time complexity. 

### **🚀 Approach**  
To efficiently determine the first missing positive integer:
1. **In-place Hashing**: We will treat the input array itself as a hash map. The idea is to place each number in its correct position (i.e., the number `x` should be in the index `x-1`), whenever possible.
2. **Iterate through the array**: For each element `num`, if it is in the range `[1, n]` (where `n` is the array length) and not in its correct position, we swap it with the element at its correct index.
3. **Second Pass**: After rearranging, in a second pass, we check the arrangement to determine the first index that does not have the expected integer.
4. **Result**: The first index where the expected integer is missing gives us the first missing positive integer.

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n) - We traverse the array a constant number of times.
- **Space Complexity**: O(1) - We do the operations in-place without using extra space for another array.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();
        
        for (int i = 0; i < n; i++) {
            // Place nums[i] in its correct position if possible
            while (nums[i] > 0 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                swap(nums[i], nums[nums[i] - 1]);
            }
        }
        
        // Identify the first missing positive
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }
        
        return n + 1; // If all numbers from 1 to n are present
    }
};  
```
