# **LeetCode Copy List with Random Pointer**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The problem involves creating a deep copy of a linked list where each node contains not only a next pointer to the next node in the list but also a random pointer that can point to any node in the list or NULL. This requires careful handling of pointers to ensure that we replicate the structure of the original list accurately.  

### **🚀 Approach**  
1. **Hash Map for Node Mapping**: Use a hash map to map each original node to its corresponding copy. This way, we can ensure that random pointers are copied correctly.
2. **First Pass**: Iterate through the original list and create copies of the nodes. Store these copies in the hash map with the original nodes as keys.
3. **Second Pass**: Iterate through the original list again to assign the `next` and `random` pointers of the copied nodes using the hash map.
4. **Return the Head of the Copied List**: After setting up all the necessary pointers, return the head of the new list.

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n) where n is the number of nodes in the linked list, since we traverse the list twice.  
- **Space Complexity**: O(n) for the hash map used to store mappings from original nodes to their copies.

## **2️⃣ LeetCode Submission**  
```cpp
class Node {
public:
    int val;
    Node* next;
    Node* random;
    Node(int _val) : val(_val), next(nullptr), random(nullptr) {}
};

class Solution {
public:
    Node* copyRandomList(Node* head) {
        if (!head) return nullptr;

        unordered_map<Node*, Node*> map;
        Node* curr = head;
        
        while (curr) {
            map[curr] = new Node(curr->val);
            curr = curr->next;
        }

        curr = head;
        while (curr) {
            map[curr]->next = map[curr->next];
            map[curr]->random = map[curr->random];
            curr = curr->next;
        }

        return map[head];
    }
};
```
