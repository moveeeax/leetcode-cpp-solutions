# **Jump Game II**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The problem presents a game where you need to reach the last index of an array starting from the first index. Each element in the array represents the maximum jump length you can take from that position. The goal is to determine the minimum number of jumps required to reach the last index.  

Given this setup, the key observations are:  
- If you're at index `i` and your current value is `nums[i]`, you can jump up to `i + nums[i]`.
- The jumps are dependent on each index, so we need a systematic way to determine the fewest jumps dynamically.

### **🚀 Approach**  
The approach involves:
1. **Greedy Algorithm:** Instead of generating all possible jumps and checking them, iteratively update the furthest reachable index you can achieve with each jump. This allows us to determine when we need to make a jump and how far we can go in the least number of jumps.
2. **Initialization:** Start from the first position and initialize jumps and range markers that help track how far you can reach and when to increment the jump count.
3. **Iterate the array:** Loop through each index, updating the furthest reachable index. When you reach the end of the current range of jumps, increment the jump count and reset the range.

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n) - We traverse the list at most twice due to checking and updating reachable indices.
- **Space Complexity**: O(1) - We use a constant amount of space.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    int jump(vector<int>& nums) {
        int jumps = 0, currentEnd = 0, farthest = 0;
        for (int i = 0; i < nums.size() - 1; i++) {
            farthest = max(farthest, i + nums[i]); // Update the farthest we can reach
            if (i == currentEnd) { // Time to make a jump
                jumps++;
                currentEnd = farthest; // Update the end of our current jump
            }
        }
        return jumps;
    }
};
```
