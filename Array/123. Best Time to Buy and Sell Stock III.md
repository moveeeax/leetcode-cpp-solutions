# **Best Time to Buy and Sell Stock III**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The problem requires us to find the maximum profit we can achieve by completing at most two transactions (buying and selling) from an array of stock prices given for consecutive days. A transaction consists of buying one day and selling on another day after the buy. We start with a balance of zero and need to optimize our buying and selling to ensure we maximize profits.

To efficiently solve this problem, we will break it down into subproblems and use dynamic programming to track the best achievable profits at different stages of transactions.

### **🚀 Approach**  
1. **Define DP Arrays**: We maintain two arrays, `buy` and `sell`, where:
   - `buy[k][i]` stores the maximum profit achievable with `k` transactions up to day `i` while holding stock.
   - `sell[k][i]` stores the maximum profit achievable with `k` transactions up to day `i` while not holding stock.

2. **Recurring Relations**:
   - For each day, if we choose to buy a stock, we should take the maximum profit achievable from the previous day plus the current price. 
     - `buy[k][i] = max(buy[k][i - 1], sell[k - 1][i - 1] - prices[i])`
   - If we choose to sell, we would similarly add the current price to the maximum profit achievable from holding the stock on the previous day:
     - `sell[k][i] = max(sell[k][i - 1], buy[k][i - 1] + prices[i])`
  
3. **Iterate Through Prices**: We will iterate through the prices array, updating our `buy` and `sell` arrays accordingly based on the defined relations.

4. **Final Output**: After processing all the prices, the maximum profit with at most two transactions will be given by `sell[2][n-1]` (i.e., selling after the last price).

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n), where n is the number of days (length of prices) since we are iterating through each day's prices.
- **Space Complexity**: O(1), since we can optimize our space usage by only storing the last transactions instead of maintaining arrays for all transactions.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        if (prices.empty()) return 0;
        
        int n = prices.size();
        vector<int> buy(3, INT_MIN), sell(3, 0); // We need only 2 transactions

        for (int price : prices) {
            for (int k = 2; k >= 1; --k) {
                buy[k] = max(buy[k], sell[k - 1] - price);
                sell[k] = max(sell[k], buy[k] + price);
            }
        }
        
        return sell[2];
    }
};
```
