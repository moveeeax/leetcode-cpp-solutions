# **Gas Station**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The problem is about determining whether we can complete a circular route starting from a gas station, given the amount of gas at each station and the cost to travel to the next one. If it's possible to complete the circuit, the function should return the starting gas station index; otherwise, it should return -1.  

### **🚀 Approach**  
1. **Initialize variables**: We need to keep track of the total gas, total cost, current gas, and a potential starting index.  
2. **Iterate through each gas station**: For each station, we add the gas available there to our current gas and subtract the cost of reaching the next station.  
3. **Check feasibility**: If at any point our current gas drops below zero, it indicates that we cannot start from the current starting index. We then reset the starting index to the next station and current gas to zero.
4. **Total checks**: After iterating through all stations, if the total gas collected is greater than or equal to the total cost, return the starting index; otherwise, return -1.

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n) - We traverse the list of gas stations once.  
- **Space Complexity**: O(1) - We only use a few variables for tracking sums and indices.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    int canCompleteCircuit(vector<int>& gas, vector<int>& cost) {
        int totalGas = 0, totalCost = 0, currentGas = 0, startIndex = 0;
        
        for (int i = 0; i < gas.size(); i++) {
            totalGas += gas[i];
            totalCost += cost[i];
            currentGas += gas[i] - cost[i];
            
            if (currentGas < 0) {
                startIndex = i + 1; // reset starting index to the next station
                currentGas = 0; // reset current gas
            }
        }
        
        return (totalGas >= totalCost) ? startIndex : -1;
    }
};
```

        for (int i = 0; i < gas.size(); i++) {
