# **LeetCode Jump Game**  

## **1пёЏвѓЈ Problem Understanding**  
### **рџ“Њ Intuition**  
The problem involves determining whether you can reach the last index of an array defined by non-negative integers, where each integer indicates the maximum jump length from that position. If you analyze the mechanics of jumping, you need to keep track of the farthest position you can reach as you iterate through the array. The goal is to check after processing each index if the farthest reachable index is at least the last index of the array.

### **рџљЂ Approach**  
1. Initialize a variable `farthest` to keep track of the maximum reachable index.
2. Loop through each index `i` in the array. Check if `i` is within the bounds of `farthest`.
3. If it is, update `farthest` to be the maximum of its current value and `i + nums[i]` (current index plus jump length).
4. If `farthest` is greater than or equal to the last index of the array, return true. 
5. If the loop completes without reaching the last index, return false.

### **вЏ±пёЏ Complexity Analysis**  
- **Time Complexity**: O(n), where n is the number of elements in the input array, since we just traverse the array once.  
- **Space Complexity**: O(1), as we use only a few extra variables for storing index values.

## **2пёЏвѓЈ LeetCode Submission**  
```cpp
class Solution {
public:
    bool canJump(vector<int>& nums) {
        int farthest = 0;
        int n = nums.size();
        for (int i = 0; i < n; ++i) {
            if (i > farthest) return false; // Cannot reach this index
            farthest = max(farthest, i + nums[i]);
            if (farthest >= n - 1) return true; // Can reach the end
        }
        return false;
    }
};
```
