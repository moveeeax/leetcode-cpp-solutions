# **LeetCode Candy**  

## **1️⃣ Problem Understanding**  
### **📌 Intuition**  
The problem is about distributing candies to children based on their ratings. The rules are that each child must receive at least one candy, and children with a higher rating than their neighbors must receive more candies than those neighbors. The goal is to find the minimum number of candies required to satisfy these conditions.

### **🚀 Approach**  
1. **Initial Setup**: Start with an array of `1`s representing that each child gets at least one candy.
2. **Left to Right Pass**: Traverse the ratings array from left to right. Whenever a child has a higher rating than the previous child, give them one more candy than the previous child.
3. **Right to Left Pass**: Traverse the ratings array from right to left. Whenever a child has a higher rating than the next child, ensure they have more candies than that child.
4. **Calculate Total Candies**: Finally, sum up all the candies to get the total needed.

### **⏱️ Complexity Analysis**  
- **Time Complexity**: O(n), where n is the number of children (or the length of the ratings array). Two passes through the ratings list are sufficient.
- **Space Complexity**: O(n) for the array that holds the count of candies for each child.

## **2️⃣ LeetCode Submission**  
```cpp
class Solution {
public:
    int candy(vector<int>& ratings) {
        int n = ratings.size();
        vector<int> candies(n, 1);
        
        // Left to right pass
        for (int i = 1; i < n; i++) {
            if (ratings[i] > ratings[i - 1]) {
                candies[i] = candies[i - 1] + 1;
            }
        }
        
        // Right to left pass
        for (int i = n - 2; i >= 0; i--) {
            if (ratings[i] > ratings[i + 1]) {
                candies[i] = max(candies[i], candies[i + 1] + 1);
            }
        }
        
        int totalCandies = 0;
        for (int i = 0; i < n; i++) {
            totalCandies += candies[i];
        }
        
        return totalCandies;
    }
};  
```
